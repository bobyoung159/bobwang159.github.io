异常处理 

一.try...catch捕获异常
就把可能出异常的代码放入catch即可

二.异常类的继承关系
 (1)每个catch块都是专门用于处理该异常类及子类的一场实例。
    在java接收到异常对象，会依次判断该异常对象是否是catch块后异常类及子类，是就调用catch块中代码处理，不是调用下一个catch块。
    所有的非正常情况分为Error和Exception。
    下面列出一些常见异常

示例代码：
    public class DivTest {
        public static void main(String[] args){
            try{
                int a=Integer.parseInt(args[0]);
                int b=Integer.parseInt(args[1]);
                int c=a/b;
                System.out.println("两数相除结果为："+c);
            }catch(indexoutofboundsexception ie){
                System.out.println("数组越界，输入参数不够");
            }catch(NumberFormatException me){
                System.out.println("数字格式异常：程序只能接收整数参数");
            }catch(ArithmeticException ae){
                System.out.println("算术异常");
            }catch(Exception e){//先捕获小异常再捕获大异常，如果大异常先被catch，在其范围里的小异常永远无法被捕获。
                System.out.println("未知异常");
            }
        }
    }

 (2)java还支持多异常捕获，不同异常用“|”分割，如catch（indexoutofboundsexception | NumberFormatException | ArithmeticException ie）。

 (3)访问异常信息
    通过异常形参可以访问异常对象的相关信息。
    方法：
    getMessage():返回详细描述。
    printStackTrace:输出跟踪栈信息。
    getStackTrace:返回跟踪栈信息。

示例代码：
    catch(IndexOutOfBoundsException ie){
        System.out.println(ie.getMessage());
        ie.printStackTrace();
    }

 (4)使用finally回收资源
    因为在遇到异常后，该语句之后的所有语句不再执行，但是物理资源（如数据库连接，磁盘文件）必须显式回收，因此提供finally块，不论之前代码如何，
 finally块中的代码总会被执行。
    同时，异常处理可以嵌套。

示例代码：
    import java.io.FileInputStream;
    import java.io.IOException;

    public class FinallyTest {
        public static void main(String[] args){
            FileInputStream fis = null;
            try{
                fis = new FileInputStream("a.txt");
            }catch (IOException ioe){
                System.out.println(ioe.getMessage());
                return ;//强制方法返回
            }finally{//fina块必须放在catch后面
                if(fis!=null){
                    try {
                        fis.close();
                    }catch(IOException ioe){
                        ioe.printStackTrace();
                    }
                }
                System.out.println("执行资源回收");
            }
        }
    }

 (5)自动关闭资源的try语句
    允许在try关键字后跟一对圆括号，在其中放入多个资源，此处资源指必须在程序结束后显示关闭的资源，try语句会自动关闭这些资源。

 示例代码：
    import java.io.*;

    public class AutoCloseTest {
        public static void main(String[] args) throws IOException{
            final BufferedReader br = new BufferedReader(new FileReader("AutoCloseTest.java"));//要求变量为final类型
            PrintStream ps = new PrintStream(new FileOutputStream("a.txt"));//没显示使用final修饰，但只要不重新赋值，该变量就是有效的final。
            try(br;ps){
                System.out.println(br.readLine());
                ps.println("咋悄无声息");
            }
        }
    }

 (6)使用throws抛出异常。
    当前方法不知道如何处理该类型异常，将异常交由上一级调用者处理，如果main方法也不知道怎么处理，也可以throws,交给jvm，打印异常的跟踪栈信息,并终止程序。
    因此，当调用带有throws的方法时，要么放入try。。。catch语句进行处理，要么放在一个有throws声明的方法中。
    需要注意的是，子类抛出的异常不允许比父类抛出的异常多。

  示例代码：
    import java.io.FileInputStream;
    import java.io.IOException;

    public class ThrowTest {
        public static void main(String[] args) throws Exception{
            test();
        }
        public static void test() throws IOException{
            FileInputStream fil=new FileInputStream("a.txt");
        }
    }

 (7)使用throw抛出异常
    当编写代码时，会遇到一些编译不报错但是与需求相违背的情况（比如下棋的地方有棋子了），这时需要程序员自己抛出异常。
    throw语句抛出的是一个异常实例，当接收到开发者自行抛出的异常，同样会中止程序，调到对应catch块。

示例代码：
    public class ThrowEcTest {
        public static void main(String[] args){
            int [] a =new int[]{1,2,3};
            try {
                if(a[1]!=10){
                    throw new Exception("该数组首位不是十");//抛出异常实例，而非异常类。
                }
            }catch (Exception e){
                System.out.println(e.getMessage());
                System.out.println("请修改数组");
            }
        }
    }

 (8)自定义异常类
    异常的类名通常包含有用的信息，通过自定义异常类明确描述异常类，用户自定义异常应继承Exceptin基类。
    通常提供两种构造器，一个是带字符串参数的构造器，一个是无参数的构造器。

示例代码：
    public class MyexpTest extends Exception {
        public MyexpTest(){ }
        public MyexpTest(String msg){
            super(msg);//通过super调用父类构造器将该参数传给异常对象的message属性。
        }
    }

泛型 

一.泛型的概念
    在使用集合时，集合会忘记其中对象的类型，这导致可能将一个错误类型放入集合中导致出现异常的情况，同时在集合内
类型为Object,取出时要进行强制类型转换，会增加复杂度。
    因此，需要使用泛型来解决这个问题，它相当于一个过滤器，指定一种类型，当试图加入对象时，会对错误类型报错，同
时，取出对象时不用进行强制类型转换。
    反省不支持数组。

示例代码：
    import java.util.ArrayList;
    import java.util.List;

    public class GeneraicList {
        public static void main(String[] args){
            List<String> strList = new ArrayList<>();//前面的<>加类型后，后面的<>就不需要加类型，这叫菱形语法，是java9的新特性。
            strList.add("asd");
            strList.add("qwerty");
            //strList.add(5);加入int就会报错
            strList.forEach(ele->System.out.println(ele.length()));
        }
    }

二.深入泛型
    1.    可以为任意的接口，类添加泛型声明，事实上可以把类（接口）的泛型近似视为类（接口）的子类（接口），当定义了String<E>时，实    
    际上定义了无数子类，为E传入不同的类型实参，就会产生一个不同的子类，在定义了泛型形参后，就可以把形参当类来使用。

示例代码：

    public class Apple<t> {
        private t info;
        public Apple(t info){
            this.info=info;
        }
        public void setInfo(t info){
            this.info=info;
        }
        public t getInfo(){
            return this.info;
        }
        public static void main(String[] args){
            Apple<String> al= new Apple<>("苹果");
            System.out.println(al.getInfo());
            Apple<Double> a2 = new Apple<>(5.67);
            System.out.println(a2.getInfo());
        }
    }

    2.  而泛型类派生子类时也有一些要注意的点：
        1.在继承时，代码不能包含泛型形参：
        public class A extends Apples<T>{}，这种就是错的。

        2.如果从Apple<String>继承子类，Apple类中使用T 类型的方法都会替换成String类型，子类在重写方法是要对应做出调整：

        public class A1 extends Apple<String>{
            public String getInfo(){
                return "子类"+super.getInfo().toString();
            }
        }

    3.  并不存在泛型类
        系统并不会为泛型生成class文件，不管是形参被传入哪种类型的实参，都是被当做同一个类处理
            List<String> l1=new ArrayList<>();
            List<String> l2=new ArrayList<>();
            System.out.println(l1.getClass()==l2.getClass());
        同时，也不允许类型参数为静态对象，静态方法声明。

三.类型通配符

    如果在定义类，方法时有一个集合形参，其元素种类不确定，该怎么定义？

    1.使用类型通配符<?>
    通配符的元素类型可以匹配任何类型，比如List<?> c,带有通配符的List代表他是任何泛型List的父类。
    需要注意的是，因为无法确定类型，所以不能向c中添加元素。

    2.设置通配符的上限
    定义三个类
    public abstract class Shape {
        public abstract void draw(Canvas c);
    }

    public class Circle extends Shape {
        public void draw(Canvas c){
            System.out.println("画在画布"+c+"上");
        }
    }

    public class Rectangle extends Shape {
        public void draw(Canvas c){
            System.out.println("画矩形zai"+c+"上");
        }
    }
        此时，虽然Circle是Shape的子类，List<Circle>并不算做List<Shape>的子类型，所以下面的drawAll方法，List<Circle>
    不可以使用。
    public class Canvas {
        public void drawAll(List<Shape> shapes){
            for(Shape s:shapes){
                s.draw(this);
            }
        }
    }
    此时，就可以使用<? extends Shape>的形式，这里的?不表示父类，而是代表所有Shape的子类型，相当于设置了一个上限。
    public class Canvas {
        public void drawAll(List<？extends Shape> shapes){
            for(Shape s:shapes){
                s.draw(this);
            }
        }
    }

    3.设置通配符的下限
        与上限对应，通过<? super 类型>可以设置下线，？代表有类型相同或类型父类。

    4.设置泛型形参的上限
        不只是通配符，还可以为泛型形参设置上限。

        public class Apple<t extends Number> {
            public static void main(String[] args){
                Apple<Integer> al= new Apple<>();
                Apple<Double> a2=new Apple<>();
                Apple<String> a3=new Apple<>();//String不是Number子类，报错
            }
        }

        在多个上限的时候，要先继承父类，在继承接口。

四.泛型方法
    1.定义泛型方法
        即在定义方法时加上一个或多个泛型形参。
        形如：修饰符 <T,S> 返回值类型 方法名（形参）{方法体}

    示例代码：
    public class GenericMethodTest {
        static <T> void fromArrayToCollection(T[] a, Collection<T> c){
            for(T o:a){
                c.add(o);
            }
        }
        public static void main(String[] args){
            Object[] oa = new Object[100];
            Collection<Object> co =new ArrayList<>();
            fromArrayToCollection(oa,co);
            //String[] oa = new String["a"];
            //Collection<String> co =new ArrayList<>();
            //fromArrayToCollection(oa,co);

        }
    }
    在使用泛型方法是不用事先传入泛型参数，编译器会自动识别。

    2.与通配符的区别
        大部分情况，通配符与泛型方法可以互换。
        基本上，当方法中泛型形参用来表示一个或多个参数之间的类型依赖关系时使用泛型方法，其他时候可以使用通配符。
        如这种要求类型相同时：static <T> void fromArrayToCollection(T[] a, Collection<T> c)。
        或有继承关系：static <T，> void fromArrayToCollection(T[] a, Collection<T> c)

    3.泛型构造器
        java也允许在构造器签名上声明泛型形参。
        这里非常容易搞混类的泛型形参和泛型构造器，应注意跟在类后面的泛型均为类的泛型参数，构造器前面的是泛型构造器。
    示例代码：
    class MyClass<E>{
        public <T> MyClass(T t){
            System.out.println("e的参数是："+t);
        }
    }
    public class GenericDiamondTest {
         public static void main(String[] args){
             MyClass<String> mc1 = new MyClass<>(5);//该行未显式指定泛型构造器的类型，可以使用菱形语法。
             MyClass<String> mc2 = new<Integer> MyClass<String>(5);//该行显式指定泛型构造器的类型，不可以使用菱形语法。
         }
    }

五.擦除
        为了保证老代码的运行，java允许带泛型声明的类不指定实际类型，默认该泛型形参是指定的第一个上限类型。
        当把一个有泛型信息的对象赋给没有泛型信息的对象是，所有的类型信息都会被扔掉，即为擦除。

    示例代码：
    class Apple1<T extends Number>{
        T size;
        public Apple1(){}
        public Apple1(T size){
            this.size=size;
        }
        public void setSize(T size){
            this.size=size;
        }
        public T getSize() {
            return this.size;
        }
    }
    public class ErasureTest {
        public static void main(String[] args){
            Apple1<Integer> a = new Apple1<>(6);
            Integer as=a.getSize();
            Apple1 b=a;                           //此时a的所有类型声明均被擦除，b只知道a的上限是Number。
            Number size=b.getSize();
            //Interger size= b.getSize();         //此时会报错
        }
    }

java集合类（四）Map集合和Collections类相关操作 

    Map用于保存具有映射关系的数据，map中有两组值，一组保存key，key不允许重复；一组保存value，value可以重复。
    Map集合与Set集合非常相似，事实上，java实现实现一个Map，再通过包装一个所有value都为null的Map实现的Set，因此Set下的常用实现类，Map
基本都有对应，如HashMap，TreeMap,SortedMap等。
1.方法
    clear（）：清空所有键值对。
    containsKey(Value):Map是否包含Key(Value).
    get(key):返回key对应的value。
    put(key,value):添加一个键值对。
    remove（key）：删除key对应的键值对。
    getKey（Value）：返回包含的key，value。

示例代码：

    import java.util.HashMap;
    import java.util.Map;

    public class MapTest {
        public static void main(String[] args){
            Map map = new HashMap();
            map.put("疯狂java讲义",109);
            map.put("疯狂ios讲义",10);
            map.put("疯狂讲义",79);
            System.out.println( map.put("疯狂ios讲义",79));
            System.out.println(map);
            System.out.println("是否包含iOS"+map.containsKey("疯狂ios讲义"));
            System.out.println("是否录入价格"+map.containsValue(99));
            System.out.println("iOS的价格"+map.get("疯狂ios讲义"));
            map.remove("疯狂讲义");
            System.out.println(map);

        }
    }

4.TreeMap实现类
    与TreeSet相似，也有自然排序和定制排序，提供方法顺序访问元素。

    示例代码：

    import java.util.TreeMap;

    class R implements Comparable{
        int count;
        public R(int count){
            this.count=count;
        }
        public String toString(){
            return "R[count:"+count+"]";//toString函数使在输出时，输出想要的名称
        }

        @Override
        public boolean equals(Object obj) {
            if(this==obj)
                return true;
            if(obj!=null&&obj.getClass()==R.class){
                R r=(R)obj;
                return r.count==this.count;
            }
            return false;
        }

        @Override
        public int compareTo(Object o) {
            R r=(R)o;
            return count>r.count?1:count<r.count?-1:0;
        }
    }
    public class TreeMapTest {
        public static void main(String[] args){
            TreeMap tm=new TreeMap();
            tm.put(new R(3),"ASD");
            tm.put(new R(5),"ASDfg");
            tm.put(new R(4),"ASDf");
            System.out.println(tm);
            System.out.println(tm.firstEntry());
            System.out.println(tm.lastKey());
            System.out.println(tm.higherKey(new R(2)));
            System.out.println(tm.subMap(new R(-1),new R(4)));//返回范围内的子Map.
        }
    }

java集合类（三）List集合和Queue集合相关操作 

List集合
    代表一个元素有序，可重复的集合，集合中每个元素都有对应的顺序索引，因此允许出现重复元素，ArrayList是其典型实现。
    1.方法
        void add(int index,Object e):添加元素到index处。
        Object get(int index):返回index处的元素。
        int indexOf(Object e):位置索引。
        Object remove（int index）:删除并返回index处的元素。
        Object set(int index,Object e):替换，返回被替换对象。

示例代码：
    import java.util.ArrayList;
    import java.util.List;

    public class ListTest {
        public static void main(String[] args){
            List books = new ArrayList();
            books.add(new String("轻量级java ee企业实战"));
            books.add(new String("疯狂java讲义"));
            books.add(new String("疯狂安卓讲义"));
            System.out.println(books);
            books.add(1,new String("ajax讲义"));
            for(int i=0;i<books.size();i++){
                System.out.println(books.get(i));
            }
            books.remove(2);
            System.out.println(books);
            books.set(2,"疯狂java讲义");
            System.out.println(books);
        }
    }

Queue类
    用于模拟队列这种数据结构，遵循先进先出的规则。
    1.方法
    add（Object e）：将指定元素加入队列尾部。
    element（）：获得头部元素。
    peek():获取头部元素。
    pool（）：获取头部元素并删除。
    2.标准实现类-PriorityQueue
        该类为较标准的队列实现类，但是保存顺序不是按加入队列的顺序，而是按队列元素大小排序，peek（）或pool（）取出的是最小的元素。

    示例代码：
        import java.util.PriorityQueue;

        public class PriorityQueueTest {
            public static void main(String[] args){
                PriorityQueue pq =new PriorityQueue();
                pq.offer(-6);
                pq.offer(-3);
                pq.offer(7);
                pq.offer(3);
                System.out.println(pq);
                System.out.println(pq.poll());
            }
        }

java集合类（二）Set相关操作 

Set集合与Collection基本相同，只不过Set不允许出现重复元素。

1.HashSet类
HashSet类是Set接口的典型实现，在使用他进行数据储存时，会通过调用该对象的hashCode（）方法，equals（）方法来判断是否与集合中元素重复。

示例代码：
    import java.util.HashSet;
    import java.util.Iterator;

    class R{
        int count;
        public R(int count){
            this.count=count;
        }
        public String toString(){
            return "r[count:"+count+"]";    //改写toString方法，是对象按规定格式输出。
        }
        public boolean equals(Object obj){
            if(this==obj){        //对象相同的情况
                return true;
            }
            if(obj!=null&&obj.getClass()==R.class){ //非空且类型一致
                R r=(R)obj;
                return this.count==r.count;
            }
            return false;

        }
        public int hashCode(){
            return this.count; //将count作为hashcode检测值，正常情况下有标准的方法
        }
    }
    public class HarshsetTest {
        public static void main(String []args){
            HashSet hs=new HashSet();
            hs.add(new R(5));
            hs.add(new R(-3));
            hs.add(new R(9));
            hs.add(new R(-2));
            System.out.println(hs);
            Iterator it=hs.iterator();
        }
    }

--------------------------------------------------------------------------------------------------------

2.LinkedHashSet类
他是HashSet类的子类，在其基础上使用链表维护元素的次序，会按照元素添加的顺序来访问集合中的元素。

示例代码：
    import java.util.LinkedHashSet;

    public class LinkedHashSetTest {
        public static void main(String[] args){
            LinkedHashSet books =new LinkedHashSet();
            books.add("dsd");
            books.add("sdsf");
            System.out.println(books);
            books.remove("dsd");
            books.add("dsd");
            System.out.println(books);//元素顺序总是于添加顺序相同
        }
    }

--------------------------------------------------------------------------------------------------------

3.TreeSet类
    该类是SortedSet接口的实现类，保证集合元素处于排序状态。    
    在原有基础上提供了如下方法：
        first():返回集合第一个函数。
        last():返回集合最后一个函数。
        higher(Object e)：返回大于指定元素的最小元素.
        lower(Object e):返回小于指定元素的最大元素。
        headSet(Object e):返回集合中小于e的子集.
        tailSet(Object e):返回集合中大于等于e的子集.

示例代码：
    import java.util.TreeSet;

    public class TreeSetTest {
        public static void main(String[] args){
            TreeSet nums= new TreeSet();//根据数值大小排序
            nums.add(5);
            nums.add(2);
            nums.add(10);
            nums.add(-9);
            System.out.println(nums);
            System.out.println(nums.first());
            System.out.println(nums.last());
            System.out.println(nums.headSet(4));//排在四前面的
            System.out.println(nums.tailSet(5));//排在五后面的
            System.out.println(nums.subSet(-3,4));//-3和4之间的
        }
    }

同时，TreeSet支持两种排序方式：自然排序和定制排序。

（1）自然排序
    调用Comparable接口的compareTo(Object e)方法，比较大小关系，对集合元素升序排序，同时判断元素相等唯一标准
也是compareTo方法是否返回0（但也要保证equals（）返回结果一致）。
    因此，在将对象存入TreeSet之前，该对象的类必须先实现Comparable接口，显然，同类才能比较大小。

示例代码：
    import java.util.TreeSet;

    class Z implements Comparable{
        int age;
        public Z(int age){
            this.age=age;
        }
        public boolean equals(Object obj){
            return true;
        }

        @Override
        public int compareTo(Object o) {
            return 1;
        }
    }
    public class TreeSetTest2 {
        public static void main(String[] args){
            TreeSet set= new TreeSet();
            Z z1=new Z(6);
            set.add(z1);
            System.out.println(set.add(z1));
            System.out.println(set);
            ((Z)(set.first())).age=9;
            System.out.println(((Z)(set.last())).age);
        }
    }

（2）定制排序
    定制排序通过使用Comparator接口中的int compare（o1,o2）方法，该方法返回正整数，代表o1>o2,返回0，代表相等，返回负数，代表o1小于o2。
    Comparator为函数式接口，可以直接使用lambda表达式。

示例代码：
    import java.util.TreeSet;

    class M{
        int age;
        public M(int age){
            this.age=age;
        }
        public String toString(){
            return "M[age:"+age+"]";
        }
    }
    public class TreeSetTest3 {
        public static void main(String[] args){
            TreeSet ts =new TreeSet((o1,o2)->{
                M m1=(M)o1;
                M m2=(M)o2;
                return m1.age>m2.age?-1:m1.age<m2.age?1:0;
            });
            ts.add(new M(5));
            ts.add(new M(-3));
            ts.add(new M(9));
            System.out.println(ts);
        }
    }

java集合类（一）Collection相关操作 

Collection是java集合框架的根接口，同时也是set,queue和list的父接口。
1.方法
    boolean add(object obj):添加一个元素。
    void clear（）：清空集合。
    boolean contains(object obj):是否包含指定元素。
    boolean isEmpty():集合是否为空。
    boolean remove(object obj):移出指定元素。

下列程序将展示如何通过上述方法操作集合元素。
    public class CollectionTest {
        public static void main(String [] args){
            Collection c = new ArrayList();
            c.add("孙悟空");
            c.add(6);
            System.out.println("number of item:"+c.size());
            c.remove(6);
            System.out.println("number of item:"+c.size());
            System.out.println("是否包含孙悟空字符:"+c.contains("孙悟空"));
            Collection books = new HashSet();
            books.add("dasfasa");
            books.add("dssdfsf");
            System.out.println("c集合是否全包含books："+c.containsAll(books));
            c.removeAll(books);
            c.clear();
        }
    }

--------------------------------------------------------------------------------------------------------

2.遍历集合的两种方法
    （1）Lambda表达式遍历集合元素
        调用forEach（Consumer action）方法,其中Consumer是函数式接口，可以试用lambda表达式来遍历。
    （2）使用Iterator遍历集合元素
        Iterator是专门用来遍历Collection中元素，也叫迭代器，并没有盛放对象的能力，在进行迭代时，并不是把集合元素传入，而是
    把集合元素的值传入，所以修改迭代变量并不会对集合元素产生影响。

下列程序将展示如何通过上述方法遍历集合。    
    public class lambda遍历集合 {
        public static void main(String[] args){
            Collection books = new HashSet();
            books.add("as");
            books.add("sd");
            books.add("df");

            books.forEach(obj->System.out.println("迭代元素："+obj));

            Iterator it = books.iterator();//iterator迭代器
            while(it.hasNext()){
                String book =(String)it.next();
                System.out.println(book);
                if(book.equals("as")){
                    it.remove();
                }
                it.forEachRemaining(obj->System.out.println("元素"+obj));
            }
            System.out.println(books);
        }
    }

3.对集合的Stream操作
    通过对stream流式api的操作，可以简单的完成一些复杂的功能。
    步骤为：使用builder（）建立对应的流 -> 调用add()添加元素 -> 调用build（）获得对应的Stream -> 调用对应的聚集方法

    示例代码：
        import java.util.stream.IntStream;

        public class IntStreamTest {
            public static void  main(String[] args){
                IntStream is = IntStream.builder()
                        .add(20)
                        .add(13)
                        .add(-2)
                        .add(18)
                        .build();
                IntStream newIs =is.map(ele->ele*ele+2);//中间方法使用后返回流
                System.out.println(is.max().getAsInt());//在对流使用末端方法后，该流被消耗。
                //System.out.println(is.min().getAsInt());
                //System.out.println(is.sum());
                //System.out.println(is.count());
                //System.out.println(is.average());
                //System.out.println("is中所有元素的平方和都大于20："+is.allMatch(ele->ele*ele>20));

            }
        }

    Stream提供了大量的方法用于聚集操作，这些方法分为中间方法（返回值为流，可以继续调用后续方法）和末端方法（使用后流被消耗）。
    常见的中间方法有：
        （1）filter（Predicate predicate）:过滤所有不符合predicate条件的元素。
        （2）sorted():用于排序。
    末端方法有：
        (1)forEach(Consumer action）:遍历所有元素，对所有元素执行action。
        (2)min(),max()
        (3)count():返回流中元素数量

    而与此同时，有专门的的stream（）方法，可以返回该集合对应的流，大大丰富了集合的功能.
    示例代码：
        import java.util.Collection;
        import java.util.HashSet;

        public class CollectionStream {
            public static void main(String[] args){
                Collection books = new HashSet();
                books.add("dasfasa");
                books.add("dssdfsfsfadwd");
                System.out.println(books.stream()
                .filter(ele->((String)ele).contains("das")).count()
                );
                System.out.println(books.stream()
                .filter(ele->((String)ele).length()>10).count());
                books.stream().mapToInt(ele->((String)ele).length()).forEach(System.out::println);
            }
        }
